# How to write an one example guide

Write an one example guide with this structure:

1. **Title**: Clear, action-oriented (e.g., "How to format X")

2. **Rules section**:
   - Number each rule (1, 2, 3...)
   - Start with the rule principle in bold or clear statement
   - Add clarifying details or sub-points below if needed
   - Keep rules concise but complete

3. **Example section**:
   - Single, comprehensive code block showing ALL rules in practice
   - Use inline comments to label which rule each section demonstrates
   - Format: `# RULE X: [restate the rule principle]`
   - Show multiple variations of the same rule when applicable
   - Make examples realistic and practical, not trivial

4. **Style**:
   - Be prescriptive (use "do this" not "you could")
   - Focus on visual patterns and consistency
   - Explain the "why" briefly when it aids understanding

The goal: Someone should be able to scan the rules quickly, then reference the annotated examples to see exactly how to apply them.

## Template:

```markdown
# How to [Action/Format/Do Something]

1. [First rule principle]. [Additional clarification or details]
2. [Second rule principle]. [Additional clarification or details]
3. [Third rule principle]. [Additional clarification or details]
4. [Fourth rule principle]. [Additional clarification or details]
5. [Fifth rule principle]. [Additional clarification or details]
6. [Sixth rule principle]. [Additional clarification or details]
7. [Seventh rule principle]. [Additional clarification or details]
