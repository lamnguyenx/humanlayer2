# How to write git commit sub-message

- Concise summaries: Brief titles followed by bullet-pointed key insights
- Focus on decisions and details: Emphasizing architectural choices (e.g., "migrated to pwd-based color generation") and technical implementations (e.g., "using MD5 hash")
- Minimal file listings: Only mentioning file modifications when essential for context, avoiding exhaustive lists
- Action-oriented language: Highlighting what changed and why, not just what files were touched
